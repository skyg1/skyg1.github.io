Sakura主题的使用前准备

YuGuang · 2020-7-18 · 次阅读

#Sakura主题的使用前准备

一、免费CDN：jsDelivr+Github 使用方法：

1.新建一个命名为cdn的存放资源的github仓库

2.在本地目录下新建空文件使用git bash here命令行下面执行命令：

 git clone 上面仓库Clone的URL(点击code弹出的可复制的地址)

将仓库中的文件克隆到本地该目录下。

3.添加或者修改文件内容

4.执行下面命令，将本地的修改后克隆仓库推到github上

git status                    //查看状态
git add .                     //添加所有文件到暂存区
git commit -m '第一次提交'     //把文件提交到仓库
git push                      //推送至远程仓库

5.release仓库，自定义版本号

进入github上的仓库，点击release发布，自定义发布版本号。

6.通过jsDelivr引用资源

使用方法：https://cdn.jsdelivr.net/gh/你的用户名/你的仓库名@发布的版本号/文件路径
例如：

https://cdn.jsdelivr.net/gh/TRHX/CDN-for-itrhx.com@1.0/images/trhx.png
https://cdn.jsdelivr.net/gh/TRHX/CDN-for-itrhx.com@2.0.1/css/style.css
https://cdn.jsdelivr.net/gh/moezx/cdn@3.1.3//The%20Pet%20Girl%20of%20Sakurasou.mp4
注意：版本号不是必需的，是为了区分新旧资源，如果不使用版本号，将会直接引用最新资源，除此之外还可以使用某个范围内的版本，查看所有资源等，具体使用方法如下：

// 加载任何Github发布、提交或分支
https://cdn.jsdelivr.net/gh/user/repo@version/file

// 加载 jQuery v3.2.1
https://cdn.jsdelivr.net/gh/jquery/jquery@3.2.1/dist/jquery.min.js

// 使用版本范围而不是特定版本
https://cdn.jsdelivr.net/gh/jquery/jquery@3.2/dist/jquery.min.js
https://cdn.jsdelivr.net/gh/jquery/jquery@3/dist/jquery.min.js

// 完全省略该版本以获取最新版本
https://cdn.jsdelivr.net/gh/jquery/jquery/dist/jquery.min.js

// 将“.min”添加到任何JS/CSS文件中以获取缩小版本，如果不存在，将为会自动生成
https://cdn.jsdelivr.net/gh/jquery/jquery@3.2.1/src/core.min.js

// 在末尾添加 / 以获取资源目录列表
https://cdn.jsdelivr.net/gh/jquery/jquery/

二、Sakura主题使用简略版（详细的请看上一篇）

1.下载Sakura主题,并且将其解压在hexo的根目录下

hexo-theme-sakura
tips:
注意是根目录，而非主题目录

2.修改hexo博客根目录下的_config配置

（1）site信息
（2）deploy信息

  1.主要设置自己的博客部署仓库地址
  2.备份仓库，为上面仓库的分支，hexo b 发布备份

3.修改Sakura主题目录下的_config配置

详见上一篇，这里要说明一下，我是直接fork了 Sakura主题作者的cdn仓库：github.com/honjun/cdn。
由于jsDeliver的使用格式固定，这样，对于里面资源文件地址修改只需更改cdn的用户名为我自己的就ok啦！
同时，对于以后的替换，我只需clone下cdn仓库，替换修改后再执行

git status
git add .
git commit -m "描述的提交信息"
git push

发布到我的cdn仓库即可完成博客的修改。

三、个人理解总结

  • 博客所有的资源文件(图片，歌曲和视频等)都是通过URL链接从别处的服务器获取使用的，可以在github自建CDN库,以jsDeliver要求的格式 https://cdn.jsdelivr.net/gh/user/repo@version/file 直接获取使用。

  • CDN库使用(github库就是用这种方式通过git更新库)：

    1. github自建一个命名为cdn的库；
    2. 本地新建文件，在文件目录下，执行命令：
      git clone cdn的clone地址信息
      之后就会出现本地cdn仓库文件.
    3. 在本地cdn仓库文件目录下，修改添加资源后，
      执行命令：
      git status
git add .   //注意最后的'.'表示上传所有文件
git commit -m "提交的备注信息"
git push
      就将本地所有文件信息上传到github库中。

  • 主题大致修改两个_config.xml（一个在博客根目录下，一个在主题目录下）和一些Page页面的Index.md文件和languages文件夹下的zn_ch.yml文件：

    1. 博客根目录_config.xml核心配置文件：
      主要注意site信息设置，theme设置和deploy设置。
    2. 主题目录_config,xml主题配置文件（大量使用jsDeliver格式的URL设置资源）：
      • site信息设置
      • cdn站点要注意设置好相应的版本
      • mune菜单下的页面设置（菜单名称，菜单链接路径，菜单图标）
        `
      1. 菜单名称显示：经由主题的语言文件里面的yml文件转换为对应中文再显示
      2. 菜单链接路径：指向source下的新建的各种Page文件夹里面的index.md
      3. 菜单图标：单独设置
        `
    3. Index.md文件使用：要严格注意空格等形式的规则，否则在hexo g不会生成public里面的同名目录下的index.html文件或者报错。
    4. hexo主题:
      1. languages文件夹：包含中文语言转换的yml文件，此yml文件还包含分类页标签页建立
2. layout文件夹：包含大量的ejs模板文件
3. source文件夹: 包含主题使用的js,cs,资源文件。
